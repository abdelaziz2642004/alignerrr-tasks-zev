 Summary of Changes

  1. Performance Optimization - Pre-compiled Regex Patterns

  Added a _CompiledPatterns class that pre-compiles all regex patterns at module load time:
  - TRAILING_REDIRECT - detects trailing redirect operators
  - SPACED_INPUT_REDIRECT - detects < < pattern
  - SPACED_OUTPUT_REDIRECT - detects > > pattern
  - CONSECUTIVE_SEMICOLONS - detects ;; outside case statements
  - TRIPLE_SEMICOLONS - detects ;;; in case statements
  - INVALID_VAR_ASSIGN - detects = value at start

  Performance Results:
  - 11,000+ validations per second
  - ~0.09ms per validation
  - Patterns compiled once at import, reused for all validations

  2. Rich Console Integration

  Added styled error output that integrates with the app's visual style:

  from zev.command_validator import print_validation_error

  result = validate_command(cmd)
  result.print_styled(console)  # or print_validation_error(result, console)

  Styled Output Features:
  - Green checkmark for valid commands
  - Red X and "Validation Error" for invalid commands
  - Dim gray labels for position and error code
  - Yellow lightbulb emoji with suggestion text
  - Works with existing Rich Console instances

  3. Test Coverage

  All 34 edge case tests pass including:
  - Escaped characters
  - Heredocs (<<, <<<)
  - Command substitution ($(), backticks)
  - Process substitution (<(), >())
  - Complex real-world commands (git, docker, awk, sed, ssh, loops, case statements)
